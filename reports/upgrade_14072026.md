## Changes Made

### Critical Fixes
1. **`src/noc_1_5_acceleation.py`** — `py5.Py5Vector.random_2D()` does not exist
   in the py5 API (confirmed against py5's live reference docs); calling it
   raised `AttributeError` at runtime. Replaced with `Py5Vector.random(dim=2)`.
2. **`CLAUDE.md`, `SETUP_AND_RUN.md`** — both instructed `pip install p5 --no-deps`
   (the old, unmaintained library), while every script in `src/` does
   `import py5` (the new, Java/JVM-backed library — not just a rename). Rewrote
   both docs: `pip install py5`, Java 17+ prerequisite, and the known
   `line-profiler` Windows install caveat, per py5's own install docs.
3. **`requirements.txt`** — was an unrelated 273-package `pip freeze` dump
   (Jupyter, matplotlib, macOS-only `pyobjc-framework-*` packages that won't
   install on Windows). Replaced with just `py5`; pip resolves its
   platform-appropriate transitive dependencies automatically.

### Redundancy & Cleanup
4. `noc_1_5_acceleation.py` / `noc_1_6_acceleationtowardsmouse.py` — removed a
   factually incorrect comment ("py5 has no built-in limit()") and replaced the
   manual speed-cap workaround with `Py5Vector.set_limit()`, which does exactly
   that.
5. `noc_2_2_applyingaforce.py` / `noc_2_3_simulatingwithmass.py` — normalized
   `Py5Vector2D(...)` to `Py5Vector(...)` for consistency with every other file
   (both are valid py5 classes; this was a style inconsistency, not a bug).
6. Removed `Notes_NatureOfCode.md` (maintainer's decision) — it duplicated
   README.md's intro, PyCharm note, and disclaimer with no distinct content.

### Discoverability

8. Fixed README's outdated "made possible by PyP5" attribution line to credit

9. Added license and last-commit badges to README

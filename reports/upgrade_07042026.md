# Upgrade Report — 07 April 2026

Generated by Claude Code (`/upgrade-repo`) after full codebase analysis.

## Project Summary

Python port of Daniel Shiffman's *The Nature of Code* (vectors, forces, oscillation).
Each file in `src/` is a standalone py5 sketch. The repo was originally written for
the old `p5` library and is being migrated to `py5`. Target audience: learners following
the book/video series.

## Changes Made

### 1. Created `reports/` directory and this report file

### 2. Created `src/README.md`
Documents naming convention, standalone sketch structure, and placeholder files.

### 3. Fixed Bug — `noc_2_6_gravitationalattraction.py` `Attractor.hover()`
`rollover = True/False` → `self.rollover = True/False`
The hover highlight never activated because a local variable was used instead of the instance attribute.

### 4. Fixed Bug — `noc_3_4_b_pendulumsimulation.py` `Pendulum.drag()`
Added `self.angle = angle` after the angle calculation.
Mouse-dragging the pendulum bob had no effect because the computed angle was never applied.

### 5. Fixed Bug — `noc_3_5_a_springs.py` module-level vector init
Moved `origin` and `bob` instantiation from module level into `setup()`.
`Vector(width/2, 0)` at module level ran before py5 initialized `width`/`height`.

### 6. Fixed docstring — `noc_I_4_customdistrbution.py`
Docstring said "y = x" but code implements `y = x*x`. Corrected to match the code.

### 7. Fixed redundant cast — `noc_I_1_randomwalker.py`
Removed outer `int()` wrapping `random.randint()` which already returns an int.

### 8. Added placeholder comments — empty sketch files
`noc_I_0_introduction.py` and `noc_2_1_whatisaforce.py` now have a comment explaining
they are intentional placeholders (concept explained verbally in the video, no code).

### 9. Migrated 14 files from `p5` to `py5`

All files below were updated:
- `from p5 import *` → `import py5`
- All bare calls prefixed with `py5.`
- `Vector(x, y)` → `py5.Py5Vector(x, y)`
- `run()` → `py5.run_sketch()`
- `strokeWeight` → `py5.stroke_weight`
- `noStroke` / `noFill` → `py5.no_stroke()` / `py5.no_fill()`
- `pushMatrix` / `popMatrix` → `py5.push_matrix()` / `py5.pop_matrix()`
- `rectMode` / `ellipseMode` → `py5.rect_mode()` / `py5.ellipse_mode()`
- `mouse_is_pressed` → `py5.is_mouse_pressed`
- `mouse_x` / `mouse_y` → `py5.mouse_x` / `py5.mouse_y`
- `frame_count` → `py5.frame_count`
- `PI` / `TWO_PI` → `py5.PI` / `py5.TWO_PI`
- `constrain`, `cos`, `sin`, `atan2`, `radians`, `dist` → `py5.*` prefixed
- `velocity.limit(n)` → manual limit using `set_mag()`
- `magnitude_sq` → `mag_sq`
- `mousePressed()` / `mouseReleased()` → `mouse_pressed()` / `mouse_released()`
- `Point(x,y)` / `distance(p1,p2)` (p5-only) → `py5.dist(x1,y1,x2,y2)`
